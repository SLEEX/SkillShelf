# PRD: SkillShelf - AI Skill 跨平台管理工具

## 介绍

SkillShelf 是一款面向个人 AI 开发者的桌面端 Skill 管理工具，解决跨平台（Claude Code、Codex、Gemini、OpenCode）使用 Skill 时遇到的共享困难、分类混乱、缺乏中文说明等核心痛点。用户可以通过图形化界面统一管理 Skill，按分类组织，并一键将指定分类的 Skill 部署到目标 AI 平台。

**技术栈：** Tauri 2.x（Rust 后端 + React/Vue 前端）+ TypeScript，打包为 Windows `.exe` 安装包。

---

## 目标

- 提供统一的 Skill 库，消除跨平台重复维护的成本
- 支持中文备注，降低 Skill 理解成本
- 通过分类系统，实现 Skill 的灵活组合与一键部署
- 分阶段交付：Phase 1 为核心 MVP，Phase 2 为云同步与高级功能

## 用户故事

### Phase 1 - MVP

---

#### US-001: 初始化本地 Skill 库存储目录
**描述：** 作为用户，我希望 App 在首次启动时选择一个本地存储目录，以便统一管理所有 Skill 文件。

**验收标准：**
- [ ] 首次启动时弹出目录选择对话框（可跳过，使用默认路径 `~/SkillShelf/skills/`）
- [ ] 存储路径可在设置页面修改
- [ ] 目录路径持久化存储在 App 配置文件中
- [ ] Typecheck/lint 通过

---

#### US-002: 导入 Skill zip 包
**描述：** 作为用户，我希望通过一键导入 zip 文件将 Skill 添加到本地库，以便快速扩充我的 Skill 集合。

**验收标准：**
- [ ] 支持拖拽或点击按钮选择 `.zip` 文件
- [ ] 解析 zip 内容并提取 Skill 元数据（名称、描述、文件列表）
- [ ] 导入时弹出对话框，允许用户选择目标分类（或放入"未分类"）
- [ ] 导入后 Skill 文件复制到本地存储目录
- [ ] 导入成功/失败均有明确的 Toast 提示
- [ ] Typecheck/lint 通过
- [ ] 在界面中验证导入流程

---

#### US-003: 查看和管理 Skill 列表
**描述：** 作为用户，我希望看到所有已导入的 Skill 列表，并能为每个 Skill 添加中文备注，以便快速理解其用途。

**验收标准：**
- [ ] Skill 列表显示：名称、原始描述（英文）、中文备注（可选）、所属分类标签、**启用/禁用状态指示**
- [ ] 每个 Skill 卡片上显示独立的启用/禁用开关（Toggle），可快速切换单个 Skill 的部署状态
- [ ] 点击 Skill 卡片可在右侧面板展开详情（文件预览、完整描述），**详情面板中提供启用/禁用控制**
- [ ] Skill 启用/禁用状态按平台独立存储（同一 Skill 可在 Claude Code 启用、在 Codex 禁用）
- [ ] 支持内联编辑中文备注，失焦自动保存
- [ ] 支持按名称搜索 Skill
- [ ] Typecheck/lint 通过
- [ ] 在界面中验证列表展示、备注编辑与单个 Skill 启用/禁用操作

---

#### US-004: 创建和管理 Skill 分类
**描述：** 作为用户，我希望自定义分类名称并将 Skill 分配到分类中，以便按用途组织我的 Skill 库。

**验收标准：**
- [ ] 支持创建分类（输入名称，可选描述和颜色标签）
- [ ] 支持重命名、删除分类（删除分类不删除 Skill 本体）
- [ ] 一个 Skill 可同时属于多个分类
- [ ] **每个分类显示启用状态指示**（如：已启用到 Claude Code、已启用到 Codex 等，带平台图标/角标）
- [ ] 分类下显示 Skill 数量角标（格式：`已启用数 / 总数`）
- [ ] 存在"未分类"默认分类（不可删除）
- [ ] Typecheck/lint 通过
- [ ] 在界面中验证分类 CRUD 操作与启用状态展示

---

#### US-005: 一键启用/禁用 Skill 分类（部署到 AI 平台）
**描述：** 作为用户，我希望选择目标 AI 平台并将指定分类的 Skill 一键部署过去，同时能精细控制单个 Skill 的启用/禁用，无需手动复制文件。

**验收标准：**
- [ ] 支持选择平台：Claude Code / Codex / Gemini / OpenCode
- [ ] 支持同时启用多个分类到同一平台
- [ ] **双层启用控制：** 分类级启用为总开关，单个 Skill 可在已启用分类下独立禁用
- [ ] 部署逻辑：仅部署"分类已启用 **且** Skill 个体已启用"的文件
- [ ] 部署动作 = 在目标平台 Skill 目录中为符合条件的 Skill 文件创建**符号链接（symlink）**，指向本地库的原始文件
- [ ] 符号链接方式为追加：不删除目标平台目录中已存在的其他 Skill（非本 App 管理的文件不受影响）
- [ ] 启用状态持久化（App 重启后保持上次配置，包括分类级和 Skill 个体级状态）
- [ ] 禁用分类时删除平台目录中该分类下所有已部署的符号链接（本地库原始文件不受影响）
- [ ] 禁用单个 Skill 时仅删除该 Skill 对应的符号链接，不影响同分类下其他 Skill
- [ ] 部署/移除操作有进度反馈
- [ ] Typecheck/lint 通过
- [ ] 在界面中验证分类级部署、单个 Skill 启用/禁用与撤销流程

---

#### US-006: AI 平台目录配置
**描述：** 作为用户，我希望配置各 AI 平台的 Skill 目录路径，以便 App 知道将文件部署到哪里。

**验收标准：**
- [ ] 设置页提供每个平台的目录路径配置项
- [ ] 内置平台默认路径（如 Claude Code：`~/.claude/skills/`）
- [ ] 支持手动选择自定义路径（文件夹选择对话框）
- [ ] 路径验证：目录不存在时给出警告（可选择自动创建）
- [ ] Typecheck/lint 通过
- [ ] 在界面中验证路径配置与验证逻辑

## 功能需求

- FR-1: App 以 Tauri 2.x 构建，发布为 Windows `.exe` 安装包；前端使用 React + TypeScript + Tailwind CSS
- FR-2: 本地数据（Skill 元数据、分类、配置）使用 SQLite 存储（通过 Tauri 的 `tauri-plugin-sql`）
- FR-3: Skill 原始文件存放在用户指定的本地目录，App 不修改原始文件，仅以副本形式管理
- FR-4: 平台部署采用**符号链接（symlink）**方式，将 Skill 文件链接到各平台目录，以追加模式操作，不影响平台目录中非本 App 管理的文件
- FR-5: 支持同一 Skill 出现在多个分类中（多对多关系）
- FR-6: 支持在同一 AI 平台下同时启用多个 Skill 分类
- FR-9: 部署采用双层启用控制——分类级为总开关，单个 Skill 可在已启用分类下独立禁用；仅"分类启用 且 Skill 启用"的文件会被部署
- FR-10: Skill 启用/禁用状态按平台独立存储，支持同一 Skill 在不同平台有不同的启用状态
- FR-7: 所有网络操作（云同步）为可选功能，离线状态下 App 完整可用
- FR-8: 界面语言：中文为主，支持中英文 Skill 描述并存显示

---

## 非目标（超出范围）

- 不提供 Skill 编辑器（仅管理，不创建/编写 Skill 内容）
- 不提供 AI 推荐 Skill 功能
- Phase 1 不包含云同步功能
- 不支持 macOS / Linux（Phase 1 仅针对 Windows）
- 不提供 Skill 市场或社区功能
- 不自动检测 AI 平台安装路径（App 启动时按各平台默认路径自动检索，用户可手动覆盖）

---

## 设计考量

- **整体风格：** 现代简洁，参考 Obsidian / Raycast 的侧边栏 + 主内容区布局
- **主色调：** 深色主题为主，支持亮色主题切换
- **核心交互：**
  - 左侧栏：分类树（每个分类带启用状态指示）+ 平台选择器
  - 主区域：Skill 卡片列表（支持列表/网格切换），每个卡片带独立启用/禁用 Toggle
  - 右侧面板：Skill 详情 + 中文备注编辑 + **启用/禁用控制开关**
- **一键部署按钮：** 在分类页面顶部显著位置，带状态指示（已部署 / 未部署）

## 技术考量

| 层级 | 技术选型 |
|------|----------|
| 桌面框架 | Tauri 2.x（Rust 后端） |
| 前端框架 | React 18 + TypeScript |
| 样式 | Tailwind CSS + shadcn/ui |
| 图标库 | [IconPark](https://github.com/bytedance/iconpark)（`@icon-park/react`） |
| 本地数据库 | SQLite（tauri-plugin-sql） |
| 构建打包 | Tauri bundler（生成 .exe 安装包） |
| 云同步（Phase 2）| WebDAV 客户端 |

**各平台默认 Skill 目录（App 启动时自动检索）：**
```
Claude Code:  ~/.claude/skills/
Codex:        ~/.codex/skills/
Gemini:       ~/.gemini/skills/
OpenCode:     ~/.opencode/skills/
```
> 各平台 Skill 文件格式基本兼容，无需转换层；若检索到目录不存在则提示用户手动配置。
